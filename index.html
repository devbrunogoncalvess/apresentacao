Diagnóstico
O Problema Atual
🖱️
Processo Manual

Seleção manual de demandas

Alguém precisa revisar todos os Work Items, identificar quais estão prontos e selecionar um a um para publicação.

⏳
Gargalo de Tempo

Atualização de status individual

Cada Work Item tem seu status atualizado manualmente para "Concluído" após a publicação, gerando retrabalho.

🔀
Risco Operacional

PRs criados manualmente

Pull Requests feitos diretamente na master sem um processo padronizado aumentam o risco de erros humanos.

📉
Baixa Rastreabilidade

Sem automação de CD

Apenas o CI está automatizado. O processo de entrega (CD) depende de ações manuais e não tem auditoria clara.

Proposta
A Solução Proposta
🚀

Pipeline Inteligente com Aprovação Humana

Automatizar tudo que pode ser automatizado — criação de PRs, atualização de Work Items e rastreabilidade — mantendo o controle humano apenas no ponto crítico: a aprovação antes do merge na master.

01

Service Hook

Gatilho automático quando um Work Item muda de estado para "Concluído".

02

Auto PR via REST API

Pipeline cria o Pull Request da branch correta para a master automaticamente.

03

Update Automático

Após o merge, o Work Item é atualizado para "Publicado" sem intervenção humana.

Arquitetura
As 3 Peças da Solução
🔔

Service Hook

Gatilho configurado no Azure DevOps que escuta mudanças de estado dos Work Items.

  • Evento: Work Item Updated
  • Filtro: State = "Concluído"
  • Ação: Dispara o pipeline de PR
  • Sem código adicional necessário
📄

Pipeline Auto-PR (YAML)

Pipeline que recebe o ID do Work Item e cria o PR automaticamente via REST API.

  • Recebe workItemId e branchName
  • Autentica via System.AccessToken
  • Cria PR com título e descrição padrão
  • Vincula o PR ao Work Item
✅

Pipeline Update WI (YAML)

Pipeline acionado pelo merge na master que atualiza o status do Work Item.

  • Trigger: push na master
  • Busca WI vinculado ao commit
  • Atualiza estado via PATCH API
  • Log auditável de cada publicação

Código
Pipeline: Criação Automática de PR
azure-pipelines-auto-pr.yml 
# Pipeline disparado via Service Hook quando WI → Concluído
trigger: none

parameters:
  - name: workItemId
    type: string
  - name: branchName
    type: string

variables:
  ORGANIZATION: 'sua-organizacao'
  PROJECT:      'seu-projeto'
  REPO:         'seu-repositorio'

stages:
  - stage: CreatePR
    jobs:
      - job: AutoPR
        pool:
          vmImage: 'ubuntu-latest'
        steps:
          - task: PowerShell@2
            inputs:
              targetType: 'inline'
              script: |
                # Configura headers de autenticação
                $headers = @{
                  Authorization  = "Bearer $(System.AccessToken)"
                  "Content-Type" = "application/json"
                }
                # Monta o body do PR com vínculo ao Work Item
                $body = @{
                  title         = "Auto PR - WI #$(workItemId)"
                  sourceRefName = "refs/heads/$(branchName)"
                  targetRefName = "refs/heads/master"
                  workItemRefs  = @(@{ id = "$(workItemId)" })
                } | ConvertTo-Json
                # Cria o PR via REST API
                $url = "https://dev.azure.com/$(ORGANIZATION)/$(PROJECT)/_apis/git/repositories/$(REPO)/pullrequests?api-version=7.1"
                Invoke-RestMethod -Uri $url -Method POST -Headers $headers -Body $body

Pipeline: Atualização do Work Item
azure-pipelines-update-workitem.yml 
# Pipeline disparado após merge na master
trigger:
  branches:
    include:
      - master

variables:
  ORGANIZATION: 'sua-organizacao'
  PROJECT:      'seu-projeto'

stages:
  - stage: UpdateWorkItem
    jobs:
      - job: UpdateWI
        pool:
          vmImage: 'ubuntu-latest'
        steps:
          - task: PowerShell@2
            inputs:
              targetType: 'inline'
              script: |
                $headers = @{
                  Authorization  = "Bearer $(System.AccessToken)"
                  "Content-Type" = "application/json"
                }
                # Busca Work Items vinculados ao commit do merge
                $commitId = "$(Build.SourceVersion)"
                $repoId   = "$(Build.Repository.ID)"
                $urlCommit = "https://dev.azure.com/$(ORGANIZATION)/$(PROJECT)/_apis/git/repositories/$repoId/commits/$commitId/workItems?api-version=7.1"
                $workItems = Invoke-RestMethod -Uri $urlCommit -Method GET -Headers $headers

                foreach ($wi in $workItems.value) {
                  # Atualiza estado do Work Item para "Publicado"
                  $bodyUpdate = @(
                    @{ op = "add"; path = "/fields/System.State"; value = "Publicado" }
                  ) | ConvertTo-Json

                  $urlUpdate = "https://dev.azure.com/$(ORGANIZATION)/$(PROJECT)/_apis/wit/workitems/$($wi.id)?api-version=7.1"
                  Invoke-RestMethod -Uri $urlUpdate -Method PATCH -Headers $headers -Body $bodyUpdate -ContentType "application/json-patch+json"
                  Write-Host "✅ Work Item #$($wi.id) → Publicado"
                }
